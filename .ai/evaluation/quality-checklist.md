# Quality Checklist

## Purpose

- Provides comprehensive quality assurance checklist
- Ensures consistent evaluation of AI-generated code
- Guides AI agents in quality-focused development
- Maintains high standards across all deliverables

## Pre-Development Checklist

### Requirements Analysis
- [ ] Requirements are clear and unambiguous
- [ ] Acceptance criteria are defined
- [ ] Constraints and limitations are identified
- [ ] Success metrics are established

### Design Planning
- [ ] Architecture follows project patterns
- [ ] Components are properly designed
- [ ] Dependencies are identified and managed
- [ ] Integration points are defined

## Development Checklist

### Code Quality
- [ ] Code follows coding standards exactly
- [ ] Naming conventions are consistent
- [ ] Functions are small and focused
- [ ] Code is well-structured and organized

### Functionality
- [ ] Requirements are fully implemented
- [ ] Edge cases are handled appropriately
- [ ] Error handling is comprehensive
- [ ] Business logic is correct

### Performance
- [ ] Code meets performance benchmarks
- [ ] Resource usage is optimized
- [ ] No memory leaks or inefficiencies
- [ ] Scalability considerations are addressed

## Post-Development Checklist

### Testing
- [ ] Unit tests are comprehensive
- [ ] Integration tests cover key scenarios
- [ ] Test coverage meets requirements
- [ ] All tests pass consistently

### Documentation
- [ ] Code is well-documented
- [ ] API documentation is complete
- [ ] Usage examples are provided
- [ ] Change documentation is updated

### Security
- [ ] Security best practices are followed
- [ ] Input validation is implemented
- [ ] Authentication/authorization is correct
- [ ] No security vulnerabilities exist

### Review and Validation
- [ ] Code review is completed
- [ ] Functionality is validated
- [ ] Performance is verified
- [ ] User acceptance is confirmed

## AI Agent Guidelines

- Complete all checklist items before marking work complete
- Use checklist as guide during development process
- Document any checklist exceptions with justification
- Continuously improve checklist based on experience
